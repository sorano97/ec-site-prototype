## データ設計

以下は、今回の納品物を指定サイトへ実装する際の最小 ER 図です。`product` を中心に、`category`、`seller`、`price_snapshot` を分けると更新運用がしやすくなります。

```mermaid
erDiagram
    CATEGORY ||--o{ PRODUCT : classifies
    SELLER ||--o{ PRODUCT : sells
    PRODUCT ||--o{ PRICE_SNAPSHOT : has
    PRODUCT ||--o{ REVIEW_SNAPSHOT : has

    CATEGORY {
      string category_id PK
      string category_name
    }

    SELLER {
      string seller_id PK
      string seller_name
      string seller_type
    }

    PRODUCT {
      string asin PK
      string product_name_ja
      string amazon_url
      string category_id FK
      string seller_id FK
      string feature_summary
      boolean is_active
    }

    PRICE_SNAPSHOT {
      string snapshot_id PK
      string asin FK
      int price_jpy
      datetime checked_at_jst
      string note
    }

    REVIEW_SNAPSHOT {
      string snapshot_id PK
      string asin FK
      float rating
      int review_count
      datetime checked_at_jst
    }
```
